Roboto 🤖 a chatbot designed to represent Marius

We are heading towards exciting times when a significant portion of “human” communication will be conducted by robots. This is my modest attempt at contributing to that future.

The project is built on the Django framework with JavaScript used on the frontend. Chatting can be conducted either through messaging or a voice-to-text feature. AJAX is utilized to send the transcribed text to the server and receive a response without refreshing the page. The actual voice transcription is carried out using the Web Speech API. User responses are sent to the OpenAI Assistant API, which has all relevant context about Marius and custom instructions for responding. The project is hosted on a website server with production done localy, pushed to GitHub and then pulled to the server. Additional safety features include transcription being done on the client side, data stored only in the current browser tab session, and the API keys fetched from environment variables.

Scraping, analyzing and visualizing sentiment of 1k Amazon reviews

I was tasked with creating a project management plan and conducting product analysis for an ecommerce product launch. This presented a perfect chance to do some web scraping to figure out customer profile based on review data.

With Amazon recently cutting down the access to review history it took some Python scripting gymnastics to scrape 932 unique reviews. I did the exploratory analysis using ChatGPT 4 and analyzed each review individually with Google Sheets ChatGPT 3.5 integration. For data formatting used the Pandas library and for the visual stuff turned to Seaborn for graphs.

Interestingly enough originally main product photos featured a man and his kid. However the data tells a different story, there are about 1/3 more women reviewers than men. Also around 26% of reviewers mentioned having a pet compared to just 9% who mentioned having kids. Thus featuring a pet-owning woman would target larger customer segment leading to increase in sales.
